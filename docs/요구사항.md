# AgentGo Biz MVP 요구사항 정의서

## 0. MVP 범위 및 전제조건

### MVP 개발 원칙
본 문서는 AgentGo Biz의 **MVP(Minimum Viable Product)** 단계 요구사항을 정의한다.
- 핵심 AI 분석·자동화 가치를 최단 시간 내 검증하는 것이 목표
- ERP, POS 실시간 API 연동은 **MVP 이후(Phase 2)** 구현
- MVP에서 POS 직접 변경 실행은 **시뮬레이션/미리보기** 수준으로 제공
- **근거 중심 분석(Evidence-based):** 모든 AI 제안 및 분석 결과에는 기반이 되는 Raw 데이터 지표와 기간을 명시하여 신뢰성 확보

### 데이터 투입 방식 (MVP)
> **모든 원천 데이터는 CSV/Excel Raw 파일 업로드 방식으로 투입된다.**

| 데이터 종류 | 투입 방식 | 담당 주체 | 주기 |
|---|---|---|---|
| 매출/거래 데이터 | CSV/Excel 수동 업로드 | 본사 or 슈퍼바이저 | 일간 |
| 메뉴/원가 마스터 | CSV/Excel 수동 업로드 | 본사 운영 담당자 | 변경 시 |
| 고객/포인트 데이터 | CSV/Excel 수동 업로드 | 본사 마케팅 담당자 | 주간 |
| 리뷰 데이터 | CSV/텍스트 수동 업로드 | 본사 or 슈퍼바이저 | 주간 |
| 날씨/이벤트 데이터 | 외부 공개 API 자동 수집 | 시스템 자동 | 실시간 |

### Phase 구분
| Phase | 범위 | 설명 |
|---|---|---|
| MVP (현재) | Raw 파일 업로드 기반 분석·자동화 | 수동 업로드 → AI 분석 → 액션 제안 |
| Phase 2 | POS/ERP/도도포인트 실시간 API 연동 | 커넥터 기반 자동 수집 |
| Phase 3 | POS 직접 실행 명령 | 자연어 명령 → POS 실시간 반영 |

## 1. 사용자 역할(Persona) 정의

### 1-1. 본사 운영/마케팅 담당자 (HQ Admin · Marketer)
| 항목 | 내용 |
|---|---|
| 주요 업무 | 전체 가맹점 성과 관제, 마케팅 캠페인 설계·실행, 공지·프로모션 배포, Raw 데이터 업로드 관리 |
| 핵심 페인포인트 | 수백 개 매장 데이터 수동 취합, 프로모션 효과 검증 불가, 이미지 공지 현장 반영 누락 |
| 기대 가치 | 리스크 매장 즉시 파악, 캠페인 ROI 자동 검증, 공지 이행 현황 실시간 확인, 데이터 업로드 후 즉시 인사이트 |
| 접근 범위 | 전체 매장 |

### 1-2. 가맹점 점주 (Store Owner)
| 항목 | 내용 |
|---|---|
| 주요 업무 | 매장 일일 운영, 매출·마진 관리, 직원 관리, 본사 공지 이행 |
| 핵심 페인포인트 | 복잡한 리포트 해석 어려움, 매출 하락 원인 모름, 본사 공지 놓침, 데이터 분석 역량 부족 |
| 기대 가치 | 아침에 "오늘 할 3가지"를 1분 내 확인, 자연어로 원인 즉시 질문, 공지 자동 요약 수신 |
| 접근 범위 | 본인 담당 매장 1개 (또는 복수 매장 점주인 경우 담당 매장) |

### 1-3. 슈퍼바이저 (Supervisor)
| 항목 | 내용 |
|---|---|
| 주요 업무 | 다수 가맹점 담당, 현장 방문·지도, 이상 매장 조기 개입, 본사-가맹점 간 커뮤니케이션 중개 |
| 핵심 페인포인트 | 담당 매장 개별 접속해야 비교 가능, 방문 전 준비 자료 수동 작성, 점주 액션 이행률 추적 불가, 리스크 매장 조기 식별 어려움 |
| 기대 가치 | 담당 매장 전체를 한 화면에서 비교, 방문 전 AI 분석 리포트 자동 생성, 점주 액션 이행률 추적, 이상 매장 조기 경보 수신 |
| 접근 범위 | 담당 배정된 가맹점 그룹 (구역 단위) |

## 2. 핵심 기능 요구사항

### [A] 수익성 및 메뉴 최적화
- **A1** 메뉴 원가-판매가 마진 계산: 업로드 데이터 기반 실질 마진율 산출, 원가 상승 시 경보 및 권장가 제안
- **A2** ABC 메뉴 분류: 판매량 × 마진율 교차 분석으로 유지/강화/정리 메뉴 분류 및 전략 제안
- **A3** 객단가(P)/객수(Q) 분해: 매출 변화 원인을 P 요인/Q 요인으로 진단하고 경영 언어로 해석
- **A4** 교차 판매(Cross-selling) 추천: 동시 구매율 높은 메뉴 조합 분석 및 POS 버튼 배치 제안 (MVP: 제안 문서 형태)
- **A5** 시간대별 세트 제안: 비피크 시간대 객단가 향상 전략 및 타임 세트 구성 제안
- **A6** 결제 수단별 순수익 계산: 카드/포인트/현금 수단별 수수료·할인 제외 실질 수익 비교

### [B] 고객 세그먼트 및 마케팅
- **B1** RFM 세그먼트: 업로드된 고객 데이터 기반 VIP/우수/이탈우려 고객 자동 분류
- **B2** 이탈 징후 감지: 방문 주기 지연 고객 감지, 위험 점수 산출, 복귀 오퍼 설계
- **B3** 고객 LTV 계산: 누적 결제 기반 생애가치 추정 및 상위 10% VVIP 타겟팅
- **B4** 리뷰 감성 분석: 긍정/부정 분류, 주요 키워드 추출, 맞춤형 대응 초안 생성
- **B5** 날씨·이벤트 연동 프로모션: 외부 공개 API + 업로드 과거 실적 결합 전략 제안
- **B6** 첫 방문 → 재방문 전환율 분석: 신규 고객의 2회차, 3회차 전환율 추적

### [C] 운영 효율 및 자동화
- **C1** 공지 이미지 OCR: 이미지 공지 텍스트 추출 → 핵심 요약 → 필수 이행 체크리스트 자동 생성
- **C2** 자연어 질의(QnA): "이번 달 주말 저녁 매출 하락 원인 알려줘" 형태의 AI 분석 질의
- **C3** 자연어 POS 변경 제안 (MVP 시뮬레이션): "A세트 가격 500원 인상" 입력 시 마진·매출 영향 시뮬레이션 제공 (실제 POS 적용은 Phase 2)
- **C4** 모닝 브리핑: 전일 실적/오늘 예측/필수 공지 자동 요약 (매일 오전 생성, 음성 옵션)
- **C5** 멀티 에이전트 협업: 분석→전략→실행 단계 자동 연결, 고위험 액션은 사람 승인 개입
- **C6** 이상 결제/누수 감지: 취소 급증·비정상 할인·포인트 이상 자동 알림 및 조치 권고

### [D] Raw 데이터 관리
- **D1** 데이터 파일 업로드: 매출/원가/고객/리뷰/재고/인력 **6타입 탭** 전환 UI, **드래그앤드롭** 업로드 영역, 매장·기간 매핑, 탭별 필수·권장 컬럼 안내 인라인 패널 제공 (CSV/XLSX 최대 50MB)
- **D2** 업로드 데이터 미리보기 및 검증: 파싱 결과 미리보기·데이터 품질 점수(0~100) 제공. **품질 점수 70점 미만 시 업로드 적용 차단**; 오류 행·필드·사유(누락/불일치/형식오류)와 영향도(무시 가능/처리 필요) 목록 표시
- **D3** 업로드 이력 관리: 파일명·종류·매장·기간·상태(완료/오류/처리중)·품질점수·업로드자·시각 8컬럼 테이블, 오류 건 재처리 및 오류 상세 다운로드
- **D4** 데이터 기간·매장 매핑: 업로드 파일에 매장ID·기간 정보 수동 매핑 설정; 기간 중복 시 덮어쓰기 확인 모달 필수
- **D5** 업로드 파이프라인 단계 시각화: 업로드 완료 후 정규화→KPI집계→마진가드재계산→RFM재산출→이상탐지→브리핑예약 6단계를 완료/처리중/대기중 상태로 실시간 표시; 완료 시 알림 수신 옵션 제공
- **D6** AI 데이터 충분성 및 콜드스타트: 분석 실행 전 데이터 볼륨·품질 검증; 부족 시 "콜드스타트" 모드로 진입하여 가능한 분석 범위 안내 및 추가 업로드 가이드 제공

### [E] 본사 관제 (HQ Admin)
- **E1** 전체 매장 리스크 Heatmap: 취소율/이상 할인/포인트 누수 **3개 세부 지표**별 Progress bar로 매장 위험도 시각화, 종합 등급(🔴위험/🟡주의/🟢정상) 표시
- **E2** 프로모션 ROI 검증: 전후 매출·순이익·ROI·통계적 유의성(p값) 비교; 기여요인 분해 및 다음 캠페인 개선 추천 포함
- **E3** 멀티 에이전트 관제: **에이전트 상태 / 워크플로우 / 데이터 관리 / 리스크 관제 4탭 구조**. 각 에이전트 헬스 점수(0~100)·마지막 실행 시각·오늘 처리 작업 수를 30초 주기로 갱신; 활성/주의/장애 상태 배지; 워크플로우 단계별 스텝 트래커 제공
- **E4** 공지 이행 현황: 전체 매장의 본사 공지 체크리스트 완료 현황 추적; 미완료 매장 일괄 리마인더 발송 및 SV 알림 연계
- **E5** 상단 KPI 요약 카드 4개: 활성 에이전트 수 / 주의 에이전트 수 / 처리 중 워크플로우 수 / 승인 대기 액션 수 실시간 표시

### [F] 슈퍼바이저 전용
- **F1** 담당 매장 성과 현황판: **상단에 위험/주의/정상 매장 수 KPI 카드 3개** 표시(각각 색상 강조), 하단에 담당 가맹점 전체 KPI(매출증감/마진경보/이탈위험/공지이행률/위험도/마지막방문)를 위험도 순 테이블로 비교; 기간·지표 필터 제공
- **F2** 리스크 매장 우선순위 목록: 이상 탐지·성과 하락 매장을 위험도 점수(가중합산) 순으로 나열; 위험도 계산 방식 툴팁으로 투명하게 공개
- **F3** 방문 사전 분석 리포트: 특정 매장 선택 시 AI가 주요 이슈·코칭 포인트·추천 대화 주제·미이행 액션 현황을 요약한 리포트 30초 이내 자동 생성; PDF 다운로드 및 오프라인 캐시 지원
- **F4** 점주 액션 이행률 추적: 점주에게 제안된 액션의 실행/보류/무시 현황 매장별 집계; SV 현장 방문 시 완료 처리 및 신규 액션 등록 기능
- **F5** 매장 성과 랭킹: 담당 구역 내 매장 성과 순위(①②③ 번호 배지) 및 상위-하위 매장 격차 분석; 방문 일정 일괄 등록 연계
- **F6** 공지 이행 현황 (구역): 담당 매장들의 본사 공지 체크리스트 완료율 모니터링; 미완료 매장 리마인더 발송
- **F7** 리스크 에스컬레이션: 심각 매장 이슈를 본사에 보고하는 에스컬레이션 플로우; 긴급도 선택(🔴긴급/🟡일반/🟢참고) 및 자동 포함 이슈 목록 제공

### [G] 재고 관리 (월말 실사)
- **G1** 실사 KPI 요약: 총 품목 수, 정상, 부족, 과잉 상태를 4개 카드로 요약 표시
- **G2** 식자재 실사 입력: 카테고리별(육류/채소/양념 등) 탭 구조, 품목별 이론재고 vs 실재고 비교 및 차이율 자동 계산
- **G3** AI 재고 이상 감지: 손실율 임계값(8%) 초과 품목 자동 식별 및 원인 가설(보관불량/도난 등) 제안
- **G4** 월별 실사 이력: 과거 실사 결과 및 평균 손실율 추이 조회, 카테고리별 손실 분석 제공

### [H] 인력 관리 (인시 분석)
- **H1** 인력 KPI 요약: 총 인시, 인시당 매출(목표 대비), 가용 직원, 피크 충족율 요약
- **H2** 시간대별 투입 최적화: 시간대별 실매출 vs 투입 인원 비교, 목표 인시당 매출(SPLH) 달성도 시각화
- **H3** AI 인력 조정 권고: 인력 부족·과잉 구간 감지 및 근무 스케줄 조정안 제안, 기회손실 추정액 산출
- **H4** 직원 기여도 분석: 직원별 담당 역할, 근무시간, 인시당 매출 기여도 지표 제공

## 3. 비기능 요구사항

### 성능
- 집계 조회 응답: 2초 이내 (사전 집계 데이터 기준)
- 파일 업로드 처리 (10MB 이하): 2분 이내 파이프라인 완료
- AI 에이전트 자연어 응답: 15초 이내
- OCR 처리: 이미지 1장 기준 30초 이내

### 보안
- 역할 기반 접근 제어(RBAC): `hq_admin`, `marketer`, `supervisor`, `store_owner`
- 슈퍼바이저는 담당 배정된 가맹점 데이터에만 접근 가능
- 고객 개인정보(이름·연락처·결제정보) 저장 및 표시 시 마스킹 처리
- 모든 실행·변경 액션 감사 로그 기록
- 권한 외 매장 데이터 URL 직접 접근 차단

### 가용성
- 배치 처리 실패 시 자동 재시도 및 알림
- AI 에이전트 장애 시 서비스 부분 저하(degraded mode) 허용 (집계/조회 기능 유지)
- 데이터 부족(콜드스타트) 상태에서 분석 가능 범위를 명확히 안내

### 운영 관리
