# AgentGo Biz 원페이저 (Multi-Agent 기준)

## 1) 개요
- **제품명**: AgentGo Biz
- **개발 단계**: MVP (Raw 데이터 기반, 실시간 POS/ERP 연동 이전 단계)
- **대상 사용자**: 본사 운영/마케팅 담당자, 슈퍼바이저, 가맹점 점주
- **핵심 컨셉**: Raw 데이터(CSV/Excel 업로드)를 기반으로, 다중 AI 에이전트가 **분석 → 전략수립 → 실행 제안**까지 자동 연결
- **핵심 산출물**: 점주용 오늘의 3가지 액션, SV용 담당 매장 리스크 현황판, 본사용 매장 리스크/수익성 관제, 자동 쿠폰/프로모션 실행 제안

---

## 2) 배경 (페인포인트 / 제공가치)

### 페인포인트
- 데이터가 Raw 파일 형태로 분산되어 원인 분석이 느리고 수동 취합에 시간 소모
- 공지/프로모션 정책이 이미지 문서로 내려와 현장 반영 누락 발생
- 메뉴/가격/프로모션 변경이 수익에 미치는 영향 검증이 어려움
- 점주는 복잡한 리포트보다 즉시 실행 가능한 액션을 원함
- 슈퍼바이저는 담당 매장을 개별 접속해야 비교할 수 있어 이상 매장 조기 식별이 어려움

### 제공가치
- **수익성 가시화**: 원가-판매가-수수료-할인 반영한 실질 마진 추적 (Raw 데이터 업로드 즉시)
- **리텐션 자동화**: 이탈 징후 감지 후 쿠폰/포인트 오퍼 설계 및 실행
- **운영 자동화**: 이미지 공지 OCR+요약+실행체크리스트 자동 생성
- **SV 생산성**: 담당 매장 전체 한눈 비교, 방문 전 AI 리포트 자동 생성
- **멀티 에이전트 협업**: 분석 에이전트가 원인을 찾고, 전략/실행 에이전트가 즉시 액션 제안

---

## 3) 목표 (와우경험)
- 점주가 아침에 로그인하면 1분 내 **오늘의 핵심 액션 3개**를 확인하고 바로 실행
- SV가 담당 매장 현황판을 열면 **리스크 매장이 위험도 순으로 즉시 식별**
- 슈퍼바이저가 방문 매장을 선택하면 **30초 내 AI 방문 준비 리포트 생성**
- 본사 담당자가 특정 매장 문제를 질문하면 10초 내 원인/근거/권장액션 제시
- 프로모션 전후 순이익 변화가 자동 검증되어 "남는 장사" 여부를 수치로 확인
- Raw 데이터를 업로드하면 자동 파이프라인이 실행되어 분석 즉시 갱신

---

## 4) 전제조건

### 데이터 인프라 (MVP)
- **투입 방식**: CSV/Excel 파일 수동 업로드 (실시간 API 연동 없음)
- **데이터 종류**: 매출/거래, 메뉴/원가 마스터, 고객/포인트, 리뷰 (날씨/이벤트는 외부 공개 API)
- **데이터 보관**: 직영 5년, 가맹 13개월 정책 준수
- **Phase 2 예정**: POS/ERP/도도포인트 실시간 API 커넥터

### 시스템 요건
- 성능: 집계 조회 2초 이내, AI 응답 15초 이내
- 운영조직: 본사 관리자 페이지에서 에이전트 상태/실행내역/권한 통제 가능
- 보안/컴플라이언스: 개인정보 마스킹, 감사로그, 역할기반 접근제어(RBAC)
- 콜드스타트: 데이터 부족 상태에서 분석 가능 범위를 명확히 안내

### 5-3. 페이지별 와이어프레임(요약)

#### W-01 점주 홈 (`/owner/dashboard`)

```text
상단 브리핑
  - 매장명/주소 + 오늘 핵심 액션 3개(P0/P1)
액션 카드
  - [실행] [근거] [보류]
  - 근거 모달(W-02), 실행 확인 모달(W-03)
하단 영역
  - KPI 3카드, 시간대 추이, 매장/메뉴(Resource 연동) 카드
```

#### W-06 SV 구역 현황판 (`/supervisor/dashboard`)

```text
리스크 요약 카드
  - 위험/주의/정상 매장 수
매장 테이블
  - 매출증감, 마진, 이탈, 공지이행률, 리스크
  - [상세], [방문 리포트]
모달
  - 매장 상세, 방문 리포트 미리보기
```

#### W-09 본사 관제 (`/hq/control-tower`)

```text
상단 운영 KPI
  - 정상/주의 에이전트, 승인 대기, 오류 업로드
탭 구조
  - 에이전트 | 워크플로우 | 데이터 품질 | 리스크
상호작용
  - 상세 모달, 승인 확인 모달
```

#### W-10 데이터 업로드 (`/data/upload`)

```text
파일 업로드
  - 유형 탭(매출/원가/고객/리뷰), 드래그 업로드
매장 선택
  - resource/store.csv 기반 동적 목록
  - 선택 매장 대표메뉴/메뉴PDF 연동상태 표시
업로드 이력 + 오류 상세 + 파이프라인 현황
```

#### W-11 공지 OCR (`/hq/notices`) [신규]

```text
Step 1 업로드
  - 공지 이미지 업로드, 배포대상/기한 설정
Step 2 OCR 검수
  - AI 요약 + 추출 체크리스트 검수
Step 3 배포
  - 임시저장/배포 실행
```

```text
세그먼트 선택

오퍼 설계
  - AI 추천 오퍼, 할인율/메시지 설정
실행
  - 테스트 발송, 승인 요청
```

- W-04 자연어 QnA 전용 페이지: 점주 홈 내부 요약형으로만 제공, 독립 대화형 화면은 추후
